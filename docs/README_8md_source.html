1 Elma
2 ===
3 
4 Elma is an event loop and process manager for embedded and reactive systems. Elma is structured as a C++ library of classes on top of which you build your own C++ project. It keeps track of processes, finite state machines, events, and communication channels, executing them at specified frequencies. Elma was developed as part of the University of Washington's [ECE P 520 course](https://github.com/klavins/ECEP520) on advanced embedded systems programming. More information on how Elma was designed can be found in the links below:
5 - [Event loop management](https://github.com/klavins/ECEP520/tree/master/week_6)
6 - [Events and finite state machines](https://github.com/klavins/ECEP520/tree/master/week_7)
7 - [HTTP in event loops](https://github.com/klavins/ECEP520/blob/master/week_8)

37 Manual Installation
38 ---
39 
40 To install Elma without using Docker, you will need to install the following tools:
41 - [gcc](https://gcc.gnu.org/) 4.9 or an equivalent C/C++ compiler that supports at least C++14
42 - Make (if you want to use the makefile. Alternatively, you can use some other build manager)
43 - [Doxygen](http://www.doxygen.nl/)
44 - [Google Test](https://github.com/google/googletest)
45 - Neils Lohmann's JSON library: https://github.com/nlohmann/json
46 - [My fork](https://github.com/klavins/cpp-httplib.git) of [yhirose](https://github.com/yhirose)'s HTTP library
47 
48 Installation on Unix systems (e.g. Raspbian)
49 ---
50 
51 First install required packages available through apt-get
52 
53  sudo apt-get update
54  sudo apt-get install -y cmake
55  sudo apt-get install -y cppcheck
56  sudo apt-get install -y graphviz
57  sudo apt-get install -y doxygen
58  sudo apt-get install -y cmake
59  sudo apt-get install -y libssl-dev
60 
61 Next, install Google Test
62 
63  cd /usr/src
64  sudo git clone https://github.com/google/googletest.git
65  cd googletest
66  sudo mkdir install
67  cd install
68  sudo cmake ../
69  sudo make
70  sudo make install
71 
72 Next, install the json library
73 
74  sudo mkdir /usr/local/include/json
75  cd /usr/local/include/json
76  sudo curl -O -J -L https://github.com/nlohmann/json/releases/download/v3.5.0/json.hpp
77  sudo mv json.hpp json.h
78 
79 Next, install httplib
80 
81  cd /tmp
82  git clone https://github.com/klavins/cpp-httplib.git
83  sudo mkdir /usr/local/include/httplib
84  sudo mv /tmp/cpp-httplib/httplib.h /usr/local/include/httplib
85 
86 Now you should be ready to install elma:
87 
88  cd ~
89  mkdir Code
90  cd Code
91  git clone https://github.com/klavinslab/elma.git
92  cd elma
93  make
94 
95 Note that the Makefile should detect if you are on a pi by using the command
96 
97  uname -m
98 
99 which on a pi 3 model B+ returns 'armv71'. If your pi is different, you might need to change the ifeq statments in `test/Makefile` and `examples/Makefile`.
100 
101 If you would like to treat elma as a library, so you can link in other code, you should link to the relevant files in to the unversal system resources (usr) as follows:
102 ```bash
103 ln -s include /usr/local/include/elma
104 ln -s lib/libelma.a /usr/local/lib/libelma.a
105 ```
106 Then you can do
107 ```c++
108 #include "elma/elma.h"
109 ```
110 in your code and use the `-lelma` flag while linking. See the [example elma project](https://github.com/klavinslab/elma_project) for an example.
